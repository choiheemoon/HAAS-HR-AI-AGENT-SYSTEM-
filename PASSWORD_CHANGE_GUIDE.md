# 비밀번호 변경 기능 가이드

## 완료된 작업

### 1. 백엔드 API 추가 ✅
- **엔드포인트**: `PUT /api/v1/auth/me/password`
- **요청 형식**:
  ```json
  {
    "current_password": "현재 비밀번호",
    "new_password": "새 비밀번호"
  }
  ```
- **응답**: 사용자 정보 (UserResponse)
- **보안**: 현재 비밀번호 확인 후 변경

### 2. 프론트엔드 UI 추가 ✅
- **위치**: 사용자 정보 관리 페이지 (`/profile`)
- **기능**:
  - 비밀번호 변경 버튼 클릭 시 모달 표시
  - 현재 비밀번호 입력
  - 새 비밀번호 입력 (최소 6자)
  - 새 비밀번호 확인
  - 유효성 검사 (길이, 일치 여부, 현재 비밀번호와 다름)

## 사용 방법

1. **사용자 정보 관리 페이지 접속**
   - 헤더의 사용자 아이콘 클릭 → "사용자 정보" 선택
   - 또는 직접 `/profile` 접속

2. **비밀번호 변경**
   - "비밀번호 변경" 버튼 클릭
   - 현재 비밀번호 입력
   - 새 비밀번호 입력 (최소 6자)
   - 새 비밀번호 확인 입력
   - "변경" 버튼 클릭

## 유효성 검사

- 모든 필드 필수 입력
- 새 비밀번호 최소 6자 이상
- 새 비밀번호와 확인 비밀번호 일치
- 새 비밀번호는 현재 비밀번호와 달라야 함
- 현재 비밀번호가 올바른지 확인

## 보안 기능

- 현재 비밀번호 확인 필수
- 비밀번호는 bcrypt로 해싱되어 저장
- API는 인증된 사용자만 접근 가능 (JWT 토큰 필요)
