# AI Agent Workflow 실행 예시

## 개요

AI 어시스턴트는 **질문에 답변**하고 **실제 시스템 작업을 수행**할 수 있습니다.

## 작동 방식

### 1. 질문만 하는 경우
```
사용자: "채용 프로세스를 개선하려면 어떻게 해야 하나요?"

AI: [일반적인 조언 제공]
    채용 프로세스 개선을 위해 다음을 고려하세요:
    1. 명확한 직무 설명 작성
    2. 효율적인 지원자 관리 시스템 구축
    3. AI 기반 이력서 파싱 활용
    ...
```

### 2. 실제 데이터 조회 요청
```
사용자: "직원 ID 1번의 정보를 알려줘"

AI: [get_employee_info 도구 실행]
    → 데이터베이스에서 직원 정보 조회
    → 실제 데이터 반환
    
    직원 정보:
    - 이름: 홍길동
    - 사번: EMP001
    - 부서: 개발팀
    - 직책: 시니어 개발자
    - 입사일: 2023-01-15
    - 상태: 재직
    - 기본급: 5,000,000원
```

### 3. 복합 작업 요청
```
사용자: "개발팀 직원들의 평균 급여와 근태 현황을 알려줘"

AI: [list_employees 도구 실행 - 개발팀 필터]
    [calculate_payroll_info 도구 실행 - 각 직원]
    [get_attendance_summary 도구 실행 - 각 직원]
    → 데이터 수집 및 분석
    → 종합 리포트 제공
    
    개발팀 직원 현황:
    - 총 5명
    - 평균 급여: 4,500,000원
    - 평균 근무 시간: 160시간/월
    - 지각: 2회
    ...
```

## 지원하는 Workflow

### ✅ 현재 지원하는 작업

1. **직원 정보 조회**
   - ID 또는 이름으로 조회
   - 부서별 목록 조회

2. **대시보드 통계**
   - 실시간 인사 현황
   - 채용, 이직률 등

3. **급여 정보**
   - 급여 이력 조회
   - 급여 계산 정보

4. **근태 정보**
   - 근무 시간 통계
   - 지각, 결근 현황

5. **리포트 생성**
   - 이직률 리포트
   - 채용 지표 리포트
   - 급여 비용 리포트

### 🔄 향후 추가 예정

- 급여 계산 실행
- 직원 정보 수정
- 휴가 승인/거절
- 리포트 자동 저장

## 사용 예시

### 예시 1: 단순 조회
```
Q: "현재 총 직원 수는?"
A: [get_dashboard_stats 실행]
   현재 총 직원 수는 45명입니다.
```

### 예시 2: 상세 조회
```
Q: "홍길동 직원의 최근 30일 근태 현황은?"
A: [get_employee_info 실행 - 홍길동 찾기]
   [get_attendance_summary 실행 - ID 1, 30일]
   
   홍길동 직원의 근태 현황 (최근 30일):
   - 총 근무 시간: 160시간
   - 연장 근무: 10시간
   - 지각: 1회
   - 결근: 0회
```

### 예시 3: 리포트 생성
```
Q: "이번 달 이직률 리포트를 만들어줘"
A: [generate_report 실행 - 이직률]
   
   이직률 분석 리포트 (2024년 1월):
   - 총 퇴사자: 2명
   - 이직률: 4.4%
   - 부서별: 개발팀 1명, 마케팅팀 1명
   ...
```

## 기술적 세부사항

### LangChain Agent 구조
- **Tools**: 실제 작업 수행 함수들
- **Agent**: 도구 선택 및 실행 결정
- **Executor**: Agent 실행 관리

### Fallback 메커니즘
- Agent 사용 불가 시 → 기본 LLM 모드
- 도구 실행 실패 시 → 오류 메시지와 함께 일반 답변

## 제한사항

1. **읽기 전용**: 현재는 조회 작업만 지원
2. **데이터베이스 필요**: DB 세션이 있어야 도구 사용 가능
3. **OpenAI API 필요**: AI 기능 사용을 위해 API 키 필요
